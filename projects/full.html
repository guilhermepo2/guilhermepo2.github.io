Gamedev Club Prototypes

What is it: We are a Game Development Club where students from dfferent courses on Federal University of Uberlandia study together about game development. The goal is to create a full game until the end of the year, so we made many prototypes to try a lot of different things.

Four Prototypes Were Made: an action platformer with visual novel (dialogue) additions, a mobile racing game, a puzzle game and a "small world" mechanic.

My Contributions: The puzzle game and small world mechanics were solely made by me, the idea was to fast prototype a mechanic to see whether it was fun or not (it wasn't), they didn't take more than 30 minutes each one.

On the action platformer game I programmed the dialogue system and on the mobile racing I programmed the car controller (with gyroscope) and the camera.

Space Journey

What is it: Space Journey is a simple casual game based on the famous Flappy Bird, but space themed. You have to tap to help the space core avoid the obstacles, which are satellites, and enjoy its journey through space.

A point of difference that makes Space Journey not just another Flappy Bird clone is that I wanted a different feel with it, as it takes place on space I wanted more of a "floaty" feel. I did that by making movements slower, gravity weaker and tried to used sound effects on a lower frequency.

Space Journey is the first game I developed for Fourth Dimension Studio. It can currently be download on Play Store.

My Contributions: Basically, I did everything. Design, programming, art and the sound effects. If you want to know more, you can access the postmortem I wrote for the Fourth Dimension Devlog.

Tetris Clone

What is it: This is a Tetris Clone project developed for Web and Mobile on Unity, I did its Programming. This tetris implementation was based on the Udemy Course "Make a Tetris-style puzzler in Unity" by Walmer Lin with some subtle changes.

Changes includes gameplay elements such as button pressing delay (and whether they exist or not), order of input processing (player input, piece drop, check game over, etc...), I removed the "Hold" functionality and, originally, the queue had 3 pieces, I changed it to have just one.

A playable version can be found at itch.io, you can also check the devlog explaining it a bit better.

This was mostly a learning project to check if my guesses on how to write a tetris were correct and it was useful to get some practice on my weak points in Unity such as User Interface and Particles.
